====================================================
HEURÍSTICA PARA MODELADO DE BACKEND
====================================================

ANTES DE CREAR UN MODELO

1. ¿Qué representa?

    Describe el objeto del mundo real.

    Ejemplo:
        Product representa un artículo que la tienda vende.


----------------------------------------------------

2. ¿Por qué existe?

    ¿Qué problema resuelve?

    ¿Cuál es su propósito dentro del sistema?


----------------------------------------------------

3. ¿Quién lo crea?

    Usuario
    Sistema
    Administrador
    API externa
    Proceso automático


----------------------------------------------------

4. ¿Cuándo deja de existir?

    Nunca
    Se elimina
    Se archiva
    Solo cambia de estado


----------------------------------------------------

5. ¿Con quién se relaciona?

    Identificar relaciones.

    OneToOne
    OneToMany
    ManyToMany

    Dibujar las cardinalidades.


----------------------------------------------------

6. ¿Qué datos le pertenecen realmente?

    Solo colocar información propia.

    Evitar duplicar información.


----------------------------------------------------

7. ¿Qué datos obtiene mediante relaciones?

    Pensar qué datos provienen de otros modelos.

    Evitar guardar información innecesaria.


----------------------------------------------------

8. ¿Qué reglas de negocio tiene?

    Ejemplos:

    - Un pedido debe tener al menos un producto.
    - Un stock no puede ser negativo.
    - Un usuario solo puede cancelar pedidos pendientes.


----------------------------------------------------

9. ¿Qué información debe conservarse para siempre?

    Pensar en información histórica.

    Ejemplo:

    Product.price        -> cambia.

    OrderItem.unit_price -> nunca cambia.


----------------------------------------------------

10. ¿Estoy creando una entidad o una consulta?

    Antes de crear una tabla preguntarme:

    ¿Existe realmente?

    ¿Tiene identidad propia?

    ¿Tiene ciclo de vida?

    ¿O puede obtenerse mediante una consulta?
